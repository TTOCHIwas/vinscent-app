# 단일 버튼 녹음 UI 설계

작성일: 2026-07-16

## 1. 목적

현재 정상 동작하는 녹음, 업로드, 현재 녹음 교체, 재생, 알림 흐름은 유지하고 홈의 녹음 UI만 하나의 원형 버튼으로 단순화한다.

사용자가 보는 요소는 버튼 하나뿐이다. 제목, 현재 녹음 정보, 재생 시간, 안내 문구, 파형, 별도 정지·삭제 버튼, 확인창, 스낵바는 표시하지 않는다. 운영체제가 제공하는 최초 마이크 권한창과 화면에 보이지 않는 접근성 정보는 예외로 한다.

## 2. 현재 호출 경로

### 2.1 녹음과 교체

1. 홈 `HomeRecordingPanel`
2. `RecordingCaptureController.startRecording`
3. `AudioRecorder.start`
4. `RecordingCaptureController.finishGesture`
5. `AudioRecorder.stop`
6. `CoupleRecordingOverviewController.uploadCurrentRecording`
7. `CoupleRecordingRepository.uploadCurrentRecording`
8. Storage 업로드
9. `replace_current_couple_recording` RPC
10. 기존 현재 녹음을 새 녹음으로 교체하고 알림 이벤트 생성

새 파일의 Storage 업로드와 RPC가 성공하기 전까지 기존 현재 녹음은 유지된다.

### 2.2 재생

1. 홈 `HomeRecordingPanel`
2. `RecordingPlaybackController.toggle`
3. `AudioPlayer.setUrl`
4. 재생, 일시정지, 완료 후 처음부터 다시 재생

## 3. 단일 버튼 상태

버튼은 지름이 고정된 원형으로 구성하고 중앙 아이콘과 버튼 자체의 색상, 외곽 원형 진행 효과만 사용한다.

1. 현재 녹음 없음: 마이크 아이콘
2. 현재 녹음 있음: 재생 아이콘
3. 재생 중: 일시정지 아이콘
4. 녹음 준비 중: 녹음 색상과 부정형 원형 진행 효과
5. 녹음 중: 녹음 색상과 최대 15초 기준 원형 진행 효과
6. 업로드 중: 중복 입력이 차단된 버튼과 부정형 원형 진행 효과
7. 읽기 또는 편집 불가: 비활성화 색상의 버튼

상태 텍스트는 시각적으로 표시하지 않는다. 접근성 라벨은 `길게 눌러 녹음`, `녹음 재생`, `재생 일시정지`, `녹음 중`, `녹음 저장 중`처럼 현재 상태에 맞게 제공한다.

## 4. 제스처 규칙

- 짧게 누르기는 현재 녹음이 있을 때만 재생과 일시정지를 전환한다.
- 길게 누르기가 인식되면 짧게 누르기 콜백은 실행하지 않는다.
- 길게 누르기를 시작하면 재생 중인 오디오를 중단하고 녹음을 준비한다.
- 누르고 있는 동안 녹음을 계속하고 손을 떼면 녹음을 종료해 저장한다.
- 15초에 도달하면 손가락 상태와 관계없이 자동으로 녹음을 종료해 저장한다.
- 기존 녹음이 있어도 확인창 없이 재녹음한다.
- 준비 중 손을 떼면 실제 녹음이 시작된 직후 취소하고 업로드하지 않는다.
- 위로 밀어 취소하는 기존 제스처는 제거한다.
- 준비, 업로드 또는 재생 전환 처리 중에는 중복 제스처를 받지 않는다.

## 5. 오류와 피드백

- 녹음 시작과 종료는 햅틱과 버튼 상태 변화로만 전달한다.
- 권한 거부, 녹음 실패 또는 업로드 실패는 오류 햅틱 후 원래 버튼 상태로 복귀한다.
- 앱 자체 오류 문구, 스낵바, 다이얼로그는 표시하지 않는다.
- 실패한 새 녹음은 기존 현재 녹음을 변경하지 않는다.
- 디버그 로그와 내부 오류 매핑은 진단을 위해 유지한다.

## 6. 최적화

- 100ms마다 변경되는 녹음 진행 상태가 홈의 큰 패널 대신 고정 크기 버튼만 다시 그리도록 위젯 경계를 축소한다.
- 버튼을 `RepaintBoundary`로 격리해 원형 진행 애니메이션의 영향을 제한한다.
- `File.readAsBytes`가 반환한 `Uint8List`를 다시 복사하지 않는다.
- 녹음 성공, 실패, 준비 취소 후 임시 녹음 파일을 최선 노력 방식으로 삭제한다.
- `AudioRecorder`와 `AudioPlayer`는 기존 Provider 생명주기에 따라 한 번 생성하고 dispose한다.
- 네트워크 업로드와 RPC 계약은 변경하지 않는다.

## 7. 보존 범위

- 녹음 최대 길이 15초
- 마이크 권한 처리
- 현재 녹음 Storage 경로와 교체 RPC
- 상대방 녹음 알림
- 녹음 보관함의 데이터, 화면, 라우트와 저장 슬롯 기능
- 커플 보관 상태의 읽기 전용 재생

홈에서는 보관함 링크를 노출하지 않지만 기존 라우트와 기능은 호환성을 위해 유지한다.

## 9. 검증 기준

- 홈 녹음 영역에 원형 버튼 외의 시각 요소가 없다.
- 현재 녹음 유무와 재생 상태에 따라 마이크, 재생, 일시정지 아이콘이 표시된다.
- 짧게 누르기와 길게 누르기가 중복 실행되지 않는다.
- 녹음 중 버튼 색상과 외곽 진행률이 변경된다.
- 길게 누른 손을 떼면 저장되고 15초에 자동 종료된다.
- 재녹음 실패 시 기존 녹음이 유지된다.
- 오류 문구, 스낵바와 확인창이 나타나지 않는다.
- 홈 녹음 버튼 위젯 테스트, 녹음 단위 테스트, Flutter 정적 분석과 전체 테스트가 통과한다.
