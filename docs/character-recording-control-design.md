# 캐릭터 녹음 컨트롤 설계

작성일: 2026-07-17

## 1. 목적

홈의 독립된 원형 녹음 버튼을 제거하고 커플 캐릭터 자체를 녹음과 재생을 담당하는 단일 컨트롤로 사용한다.

캐릭터 편집은 홈에서 반복해서 사용하는 동작이 아니므로 설정으로 옮긴다. 기존 녹음, 업로드, 현재 녹음 교체, 재생, 보관함과 알림 흐름은 변경하지 않는다.

이 문서는 `single-button-recording-design.md`의 홈 표시와 제스처 컨트롤 부분을 대체한다. 해당 문서의 저장 계약, 오류 정책, 최대 녹음 길이와 보존 범위는 계속 적용한다.

## 2. 요구사항

- 홈에는 별도 마이크, 재생 또는 일시정지 버튼을 표시하지 않는다.
- 캐릭터를 짧게 누르면 현재 녹음이 있을 때 재생과 일시정지를 전환한다.
- 캐릭터를 길게 누르면 현재 녹음 유무와 관계없이 새 녹음을 시작한다.
- 길게 누른 손을 떼면 녹음을 종료하고 저장한다.
- 녹음 중에는 캐릭터 외곽의 빨간 원형 진행 효과로 최대 15초 진행률을 표시한다.
- 재생 중에는 캐릭터의 작은 맥박 애니메이션으로 상태를 표시한다.
- 새 현재 녹음을 처음 확인했을 때 캐릭터가 제한된 횟수만 맥박쳐 재생 가능성을 알린다.
- 업로드와 비동기 준비 중에는 외곽 진행 효과를 표시하고 중복 입력을 차단한다.
- 앱 자체의 설명 문구, 확인창과 오류 메시지는 추가하지 않는다.
- 화면에 보이지 않는 접근성 라벨과 운영체제 마이크 권한창은 유지한다.
- 캐릭터 편집 진입점은 설정의 커플 영역으로 옮긴다.
- 기존 `/home/character` 경로는 호환 경로로 유지하되 `/settings/character`로 이동시킨다.

## 3. 현재 호출 경로

### 3.1 홈 표시

1. `HomeScreen`
2. `_HomeCharacterArea`에서 `CoupleCharacterAvatar` 표시
3. `HomeRecordingPanel`에서 별도 `RecordingControlButton` 표시

캐릭터 탭은 `/home/character`로 이동하고 녹음 버튼은 녹음 상태 Provider를 구독한다.

### 3.2 녹음과 재생

1. `HomeRecordingPanel`
2. `RecordingCaptureController` 또는 `RecordingPlaybackController`
3. 기존 녹음 저장·교체 또는 재생 경로

새 설계는 1번의 표시 컴포넌트만 교체한다. 애플리케이션과 데이터 계층 호출은 보존한다.

### 3.3 캐릭터 편집

1. 홈 `CoupleCharacterAvatar.onTap`
2. `/home/character`
3. `CharacterEditorScreen`
4. 저장 또는 뒤로가기 시 `/home`으로 이동

새 설계에서는 설정 항목이 `/settings/character`를 열고 편집 완료 또는 뒤로가기는 탐색 스택 상태와 관계없이 `/settings`로 이동한다.

## 4. 설계 결정

### 4.1 홈 전용 컨트롤

공용 `CoupleCharacterAvatar`는 질문 화면에서도 사용하므로 녹음 Provider, 제스처와 애니메이션 책임을 추가하지 않는다.

녹음 기능의 Provider 연결을 담당하는 홈 전용 `HomeCharacterRecordingControl`과 순수 표시·제스처 위젯을 녹음 기능 영역에 둔다. 표시 위젯에는 정적인 캐릭터를 `child`로 전달해 100ms 진행률 갱신 때 이미지 하위 트리를 다시 만들지 않도록 한다.

### 4.2 상태 표현

1. 현재 녹음 없음: 캐릭터만 표시하고 길게 누르기만 활성화
2. 현재 녹음 있음: 캐릭터만 표시하고 짧게 누르기와 길게 누르기 활성화
3. 재생 중: 캐릭터와 외곽선이 작은 범위로 반복 확대·축소
4. 녹음 준비 중: 빨간 부정형 원형 진행 효과
5. 녹음 중: 빨간 확정형 원형 진행 효과
6. 업로드 또는 로딩 중: 중립색 부정형 원형 진행 효과와 입력 차단
7. 편집 불가: 재생할 녹음이 있으면 짧게 누르기만 허용

현재 녹음 식별자가 새 값으로 바뀌면 유한한 맥박 애니메이션을 실행한다. 재생 중 애니메이션만 반복하고 나머지 유휴 상태에서는 애니메이션 리소스를 사용하지 않는다.

### 4.3 제스처 경합

Flutter의 `GestureDetector` 탭과 길게 누르기 recognizer를 같은 표면에 연결한다. 길게 누르기가 제스처 경기에서 승리하면 탭은 취소되므로 재생과 녹음 시작이 중복 실행되지 않는다.

- 탭: 재생 가능한 현재 녹음이 있을 때만 실행
- 길게 누르기 시작: 햅틱, 현재 재생 중단, 녹음 시작
- 길게 누르기 종료: 종료 햅틱, 녹음 종료 및 저장
- 준비 중 손을 뗀 경우: 기존 컨트롤러 정책에 따라 취소
- 준비, 업로드와 재생 전환 중: 새 제스처 차단

### 4.4 설정 라우팅

- 설정의 커플 섹션에 `캐릭터 꾸미기` 항목을 추가한다.
- 정식 경로는 `/settings/character`이고 기존 `characterEditor` 라우트 이름을 유지한다.
- `/home/character`는 정식 경로로 리다이렉트한다.
- 편집 화면은 저장과 뒤로가기 모두 `/settings`로 명시 이동한다.
- 홈, 질문과 다른 캐릭터 표시 동작은 변경하지 않는다.

## 5. 레이아웃과 최적화

- 기존 홈의 세 구역 배치를 유지하면서 하단 독립 녹음 버튼 자리에 캐릭터 녹음 컨트롤을 둔다.
- 기존 중간 캐릭터 표시는 제거해 캐릭터가 중복되지 않게 한다.
- 컨트롤 크기와 진행 효과 크기를 고정해 상태 변화로 홈 레이아웃이 움직이지 않게 한다.
- 컨트롤을 `RepaintBoundary`로 격리한다.
- 캐릭터 하위 위젯은 상태 갱신과 무관한 동일한 `const` child로 유지한다.
- 재생 맥박만 반복하고 새 녹음 알림 맥박은 제한된 횟수 후 정지한다.
- 녹음 진행률을 위한 기존 100ms 주기와 Provider 생명주기는 유지한다.

## 6. 보존 범위

- 녹음 최대 길이 15초
- 마이크 권한과 오류 햅틱
- 현재 녹음 Storage 업로드와 교체 RPC
- 실패 시 기존 현재 녹음 유지
- 녹음 보관함 데이터, 화면과 라우트
- 상대방 녹음 알림
- 커플 보관 상태의 읽기 전용 재생
- 캐릭터 그림 데이터, 저장 방식과 편집 기능
- 질문 화면의 캐릭터 표시와 탭 동작

## 7. 구현 단계와 커밋

1. `docs: 캐릭터 녹음 컨트롤 설계 추가`
2. `test: 캐릭터 녹음 인터랙션 명세 추가`
3. `feat: 홈 캐릭터에 녹음 인터랙션 통합`
4. `feat: 캐릭터 편집을 설정으로 이동`
5. 검토에서 문제가 발견되면 원인별 보완 커밋 추가

## 8. 검증 기준

- 홈에 캐릭터가 한 번만 표시되고 별도 녹음 버튼과 마이크·재생·일시정지 아이콘이 없다.
- 녹음이 없을 때 탭은 실행되지 않고 길게 누르기만 녹음을 시작·종료한다.
- 녹음이 있을 때 탭은 재생하고 길게 누르기는 재생 콜백 없이 재녹음한다.
- 녹음 중 외곽 진행률과 재생 중 맥박 효과가 표시된다.
- 새 현재 녹음 맥박은 유한하게 종료된다.
- 준비와 업로드 중 중복 입력이 차단된다.
- 설정에서 캐릭터 편집으로 진입하고 뒤로가기와 저장 후 설정으로 돌아온다.
- `/home/character` 직접 진입은 새 설정 경로로 이동한다.
- 질문 화면의 공용 캐릭터 동작은 유지된다.
- 관련 위젯 테스트, Flutter 정적 분석과 전체 테스트가 통과한다.
