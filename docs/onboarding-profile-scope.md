# 온보딩 기본 프로필 범위

작성일: 2026-05-20

## 목적

소셜 로그인 직후 사용자의 기본 프로필이 없으면 홈 화면보다 먼저 온보딩을 노출한다.

## 이번 구현 범위

- 닉네임 입력
- 생일 선택
- 기본 프로필 저장
- 프로필 완료 여부에 따른 라우팅 제어

## 제외 범위

- 첫 만남일 입력
- 기념일 입력
- 커플 코드 연결
- 커플 연결 후 관계 시작일 설정
- 닉네임 중복 검사

첫 만남일은 상대방과 커플 코드로 연결된 뒤 두 사람이 공유하는 커플 정보로 다룬다.

## 정책

- 닉네임은 앞뒤 공백을 제거한 기준으로 2자 이상 8자 이하여야 한다.
- 닉네임 중복은 MVP에서 허용한다.
- 생일은 오늘 이후 날짜를 선택할 수 없다.
- 생일 선택은 달력 UI가 아니라 년, 월, 일 wheel picker를 사용한다.
- 날짜 선택 bottom sheet는 바깥 영역을 탭하면 닫힌다.
- 완료 버튼은 활성, 비활성 상태를 가진 공통 컴포넌트로 재사용한다.

## 저장 기준

`profiles` row가 존재하고 `onboarding_completed_at`이 있으면 기본 프로필 입력이 완료된 것으로 판단한다.

앱은 닉네임과 생일이 모두 입력된 시점에만 profile을 저장한다.
